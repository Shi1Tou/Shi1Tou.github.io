部署方式：cloudfare pages

在 Cloudflare 控制台关联 GitHub 仓库
登录 Cloudflare 控制台，进入 Pages 页面。
点击 创建项目 → 连接到 Git，选择你的 GitHub 仓库。
选择要部署的分支（如 main）。

部署说明（关键步骤）
KV 命名空间：确保已创建 NAV_LINKS 命名空间，并在项目设置中绑定（参考之前的步骤）。

部署方式：
本地部署：wrangler pages deploy
GitHub 集成：推送代码后自动部署（需在控制台配置环境变量和 KV 绑定）

配置构建设置：
框架预设：选择 None（因为是自定义项目）。
构建命令：留空（无需构建，直接部署静态文件和 Worker）。
输出目录：留空（默认使用根目录）


核心功能说明

默认账户登录：
首次部署后自动创建默认账户：用户名 admin，密码 default123。
登录时需要同时输入用户名和密码，两者匹配才能进入管理员模式。

密码修改：
管理员登录后可通过「修改密码」功能更新密码，需验证当前用户名和原密码。
新密码长度至少 6 位，确保安全性。

布局调整：
顶部导航栏按钮顺序：Logo → 登录按钮 → 三条杠按钮（符合「三条杠在登录按钮左侧」的要求）。
左侧分类侧边栏点击三条杠按钮弹出，包含分类列表和数量统计。

完整功能保留：
内外网链接切换、链接添加 / 删除、导入导出 JSON 书签等功能均正常可用。
公开 / 私有链接权限控制，非管理员只能看到公开链接。

使用步骤
部署项目并配置 KV 命名空间。
访问页面，点击右上角「登录」按钮。
使用默认账户登录：用户名 admin，密码 default123。
登录后建议立即修改密码（通过「修改密码」按钮）。
可开始添加、管理链接，或导入已有书签。

开始部署点击 部署项目，Cloudflare 会自动从 GitHub 拉取代码并部署。后续每次向 GitHub 推送代码，都会触发自动部署。
